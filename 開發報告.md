# 光影建築+ 開發報告
## App Development Report — Shadow & Light Architecture Simulator

---

## 一、開發動機

建築設計中，日照分析是最基礎也最關鍵的考量之一。
然而，現有的日照分析工具（如 AutoCAD、Revit 插件）往往需要高規格的電腦與軟體授權，
初學者難以在學習初期快速理解「太陽在哪裡、陰影如何落下」的直覺。

本 App 希望以行動裝置為平台，讓任何人都能在口袋裡帶著一個互動的日照模擬器，
隨時隨地理解光影與建築形體的關係。

## 二、核心功能

### 2.1 即時 3D 光影模擬
- 以 Three.js 3D 引擎渲染一個簡化的都市街道場景
- 包含 9 棟不同高度的建築物，仿照台灣常見的住宅與商辦配置
- 陰影由 DirectionalLight（平行光源）即時運算，模擬真實太陽光線
- 使用者可拖曳旋轉視角，從任意角度觀察陰影

### 2.2 太陽位置計算
採用標準天文公式（非查表，即時運算）：

```
太陽赤緯角 δ = 23.45° × sin(360/365 × (284 + N))
時角        H = (t - 12) × 15°
高度角 sin(h) = sin(φ)sin(δ) + cos(φ)cos(δ)cos(H)
方位角  從計算出的高度角逆推
```

其中：
- N = 一年中的第幾天
- t = 時間（24小時制）
- φ = 觀測地點緯度

### 2.3 太陽軌跡弧線
場景中以橘色弧線顯示當日太陽在天空中的完整移動路徑，
讓使用者直觀理解太陽軌跡如何隨季節改變。

### 2.4 陰影長度即時計算
底部面板即時顯示：
- 主建物（15m高，約5層樓）在當前時刻的陰影長度
- 計算公式：陰影長度 = 建物高度 ÷ tan(太陽高度角)
- 以紅色線段標示於場景地面上

### 2.5 多城市支援
提供台北（25°N）、台中（24°N）、台南（23°N）、花蓮（24°N）、高雄（23°N）
五個城市，讓使用者比較不同緯度對日照的影響。

## 三、技術架構

```
React Native (Expo SDK 54)
├── App.js                 主導覽
├── src/
│   ├── screens/
│   │   ├── SimulatorScreen.js   主模擬畫面（WebView + 控制面板）
│   │   └── LearnScreen.js       光影知識庫
│   └── utils/
│       ├── sunCalc.js           太陽位置天文公式
│       └── scene.js             Three.js 3D 場景 HTML
```

### 技術選型說明

| 組件 | 技術 | 原因 |
|------|------|------|
| 3D 渲染 | Three.js (WebGL) | 行動端最成熟的 3D 引擎，支援硬體加速陰影 |
| 3D 嵌入 | React Native WebView | Expo Managed Workflow 中整合 WebGL 的最佳方式 |
| 跨平台框架 | React Native + Expo | 一套程式碼同時部署 iOS/Android |
| 數值計算 | 純 JavaScript | 太陽位置公式不需第三方庫 |

### RN ↔ WebView 通訊
```
React Native 計算太陽位置
    ↓ injectJavaScript + MessageEvent
Three.js 接收 { azimuth, altitude, month, lat }
    ↓ 更新 DirectionalLight 位置、強度、色溫
3D 場景即時重渲染
```

## 四、設計重點

### 4.1 視覺精確度與教育目的的平衡
本 App 採用**簡化的幾何建築模型**，而非真實的複雜建築形體。
這樣的選擇基於以下考量：

1. **直覺理解優先**：簡單的方形建物讓使用者專注於陰影的形成，而非被建築細節分散
2. **運算效能**：行動裝置的 GPU 有限，簡化模型確保流暢的 60fps 渲染
3. **通用性**：簡化的都市街道是全球建築教育中最通用的教學場景

### 4.2 深色介面設計
控制面板採用深色主題（#0D1117），讓 3D 場景的色彩對比更鮮明，
同時減少在戶外使用時的螢幕眩光，符合建築師野外調查的使用情境。

### 4.3 科學公式透明化
底部陰影分析面板直接顯示計算公式，讓使用者不僅看到結果，
更能理解「為什麼」——這是工具設計中的教育責任。

## 五、學習到的核心概念

### 5.1 太陽幾何學（Solar Geometry）
- 赤緯角（Declination）的季節性變化：±23.45°
- 台灣緯度（22°–25°N）對正午高度角的影響：約 42°–88°
- 方位角與高度角的聯立方程組推導

### 5.2 建築日照設計原則
- 日照遮蔽分析（Shadow Analysis）的法規基礎
- 被動式設計（Passive Design）中的遮陽板計算
- 建築朝向與採光效率的關係

### 5.3 3D 電腦圖學基礎
- 座標系轉換（天文球面座標 → 笛卡兒座標 → Three.js 場景座標）
- 即時陰影運算原理（Shadow Mapping）
- 光源方向向量的數學表達

## 七、開發心得

這個 App 的開發讓我深刻理解到：**建築設計是數學、物理與空間感知的交叉點**。
太陽的軌跡不是隨機的，它遵從精確的天文公式；
而建築師的工作，就是在這個精確的宇宙規律之中，
設計出讓人感到舒適、有光、有溫度的空間。

用程式碼模擬這件事，讓我對建築設計有了新的敬意。
